Avoid storing secrets in Git in plaintext
	Do not commit actual passwords, API keys, or tokens.
	Use placeholders or encrypted secrets for demos.

Use Kubernetes Secrets for sensitive configuration
	Secrets are stored in etcd (cluster database) and can be referenced in Pods.
	Avoid hardcoding values in deployment manifests.

Inject secrets securely
	Environment variables: Simple, readable for apps.
	Mounted volumes: Can be used when apps expect file-based secrets.
	Ensure read-only volumes when mounting secrets.

Use additional tools for enhanced security 
	Sealed Secrets (Bitnami): Encrypt secrets for safe Git storage.
	External Secrets (AWS Secrets Manager): Secrets remain in an external system.

Limit access to secrets
	Use RBAC policies to restrict which namespaces and service accounts can read secrets.

Rotate secrets regularly
	Update secrets without rebuilding containers if possible.
	Avoid long-lived static credentials.
